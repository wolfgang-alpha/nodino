i want to build a note taking agent. maybe more like a second brain agent. an agent based on devstral-small-2 and whisper speech to text + text to speech. the idea is to have a conversation with the agent in natural language. in the background the agent writes important information to a database (sql) where also meta information about the conversation is stored (such as time). i am not yet sure if the data should also be available in simple markdown format so that the user can modify it also directly. but i definetely want the agent to have the most simplistic webui using an nginx docker container. the development language shall be scripttype, but just for the frontend. the webui shall initially just have a blank screen, with a spherical button in the bottom middle of the screen that starts the conversation. by pressing, the user indicates that he starts talking and initiates the whisper speech to text conversion process. the process is stopped by the user saying, with a short pause bevore, "over". this makes the flow of the conversation less ambiguous for both the user as well as the internal logic of the software. the backend shall be written in go btw. when the agent responds to the users instructs/querys it always trys to print relevant and related data to the screen in order to mitigate the flow of information. one showing is worth a hundred tellings, but the best is visual and audio information. at least in my opinion ;) . the agent does not display sql commands it runs in the background by default. it does so only when prompted. the same goes for the users request and the agents answer - the webui should not show this info. just actual data that is written or retrieved to/from the database. as the conversion progresses, the information displayed at first may become obsolete and is replaced by new infos. the old information slowly "fades out", ie. , it slowly becomes more and more translucent until it vanishes fully. if lots of querys come in a row. the newest infos are flowing in from the bottom. the old ones fade out and are pushed out to the top. sometimes beeing pushed out is faster than the fade out. however, there is always an interesting aesthetic about this process. it tries to imitate the abstract relations of a conversation as time progresses. every major component of this project is dockerized. take the project at /home/me/ponder for reference as it is very similiar.
